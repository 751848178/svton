# 📚 内容分类体系说明

## 核心概念：分类 vs 类型

### 🎯 内容类型 (contentType) - 功能维度

**固定的4种类型**，决定内容的**功能用途**：

| 类型 | 英文     | 图标 | 说明                     | 示例                             |
| ---- | -------- | ---- | ------------------------ | -------------------------------- |
| 指南 | guide    | 📖   | 教程、说明、指南类内容   | 小区健身房使用指南、垃圾分类教程 |
| 活动 | activity | 🎉   | 社区活动、聚会、集体事项 | 周末亲子烘焙活动、社区运动会报名 |
| 互助 | help     | 🤝   | 求助、帮助、物品借用     | 求借儿童自行车、求推荐家政服务   |
| 分享 | share    | 💡   | 经验分享、二手转让、推荐 | 育儿经验分享、闲置物品转让       |

**特点**:

- ✅ 系统预设，不可修改
- ✅ 每个内容必须选择一种类型
- ✅ 类型决定了内容的**交互方式**和**展示形式**

---

### 🏷️ 分类 (category) - 主题维度

**可自定义**，决定内容的**主题领域**：

| 分类     | 图标 | 说明                 |
| -------- | ---- | -------------------- |
| 社区生活 | 🏘️   | 物业通知、小区设施等 |
| 健康运动 | 🏃   | 健身、跑步、瑜伽等   |
| 美食烹饪 | 🍳   | 美食、烹饪、餐厅推荐 |
| 育儿教育 | 👶   | 育儿、教育、亲子活动 |
| 宠物相关 | 🐕   | 宠物养护、宠物社交   |
| 兴趣爱好 | 🎨   | 摄影、绘画、音乐等   |

**特点**:

- ✅ 可在后台自由创建、修改、删除
- ✅ 每个内容必须选择一个分类
- ✅ 分类决定了内容的**主题归类**和**筛选维度**

---

## 🔄 关系类型

**交叉关系（非从属关系）**

一个**类型**可以包含多个**分类**，一个**分类**可以包含多个**类型**。

### 示例矩阵

|             |     社区生活     |     健康运动     |   美食烹饪   |    育儿教育    |   宠物相关   |
| ----------- | :--------------: | :--------------: | :----------: | :------------: | :----------: |
| **📖 指南** | 小区门禁使用说明 | 健身器材使用指南 | 烘焙入门教程 | 新生儿护理指南 | 养狗注意事项 |
| **🎉 活动** |   业主大会通知   |   晨跑小组招募   |  周末烘焙课  |   亲子运动会   |   宠物聚会   |
| **🤝 互助** |   求推荐装修队   |   求羽毛球搭子   |  求分享烤箱  |  求推荐幼儿园  |  代遛狗服务  |

## 🎨 筛选和展示逻辑

### 移动端首页展示

**按类型分Tab**:

```
[📖 指南] [🎉 活动] [🤝 互助] [💡 分享]
```

用户点击"活动"Tab，看到的是**所有类型为activity的内容**，不论分类。

**按分类筛选**:

```
分类: [全部] [社区生活] [健康运动] [美食烹饪] ...
```

选择"健康运动"，看到的是**该分类下所有类型的内容**。

### 组合筛选

用户可以同时选择：

- 类型 = "活动"
- 分类 = "健康运动"

结果显示：健康运动分类下的所有活动（如"晨跑小组招募"）

## 🔧 管理后台创建流程

### 步骤 1: 配置分类（一次性）

访问 **分类管理** 页面，创建所需分类：

```
1. 点击"新建分类"
2. 填写：
   - 名称：健康运动
   - 编码：health_sports
   - 图标：🏃
   - 排序：1
3. 保存
```

### 步骤 2: 创建内容（日常）

访问 **内容管理** → **创建内容**：

```
1. 基本信息：
   - 标题：小区健身房使用指南

2. 分类选择：
   - 类型：📖 指南       ← 功能维度
   - 分类：健康运动      ← 主题维度

3. 内容详情：
   - 摘要：介绍健身房使用方法
   - 正文：详细说明...

4. 发布设置：
   - 状态：发布/草稿
```
