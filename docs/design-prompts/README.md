## 🎯 设计基准（2025年11月）

### 标准规范

```
设计稿尺寸: 375×812pt
设计基准: iPhone 16 (393×852pt)
设计倍率: @2x
安全区域: 顶部59px, 底部34px
```

### 核心设计参数

```
品牌色: #1890FF
渐变: linear-gradient(135deg, #1890FF, #0077FF)
背景: #F7F8FA (页面) / #FFFFFF (卡片)
圆角: 12px / 16px
间距: 基于8px网格
字号: 30/28/26/24/22px
阴影: 0 2px 8px rgba(0,0,0,0.04)
```

## 📋 页面概览

### 01. 登录页 ✅

```
页面路径: pages/login/index
页面类型: 独立页面（无TabBar）
核心功能: 手机号登录 + 微信一键登录
布局方式: 上下分层（Logo区 + 表单区）
特殊要求: 考虑Dynamic Island
```

**关键组件**:

- Logo图标 (120×120)
- 手机号输入框
- 验证码输入框
- 登录按钮（渐变）
- 微信登录按钮（绿色边框）
- 用户协议复选框

**交互状态**: 5种

- 输入框: 默认/聚焦/填写/错误
- 按钮: 默认/点击/禁用/倒计时

---

### 02. 首页（内容列表） ✅

```
页面路径: pages/index/index
页面类型: TabBar页面
核心功能: Tab切换 + 内容列表展示
布局方式: Tab栏 + 滚动列表
特殊要求: 下拉刷新 + 上拉加载
```

**关键组件**:

- Tab栏（推荐/关注/最新）
- 内容卡片（封面/标题/摘要/作者/标签）
- 互动数据（浏览/点赞/评论）
- 空状态插图

**交互状态**: 6种

- Tab: 默认/选中（下划线动画）
- 卡片: 默认/点击
- 刷新: 下拉/上拉/加载中

## 📐 统一规范

### 页面类型分类

#### TabBar页面（4个）

```
首页、发布页、分类页、我的页
特点: 底部有导航栏
安全区域: TabBar自动处理，无需额外padding-bottom
```

#### 独立页面（2个）

```
登录页、详情页
特点: 无底部导航栏
安全区域: 需要考虑底部 env(safe-area-inset-bottom)
```

## 🎬 通用交互规范

### 点击反馈

```
卡片: 背景变为#F7F8FA, 缩放98%
按钮: 缩放95%, 300ms过渡
图标: 透明度0.7
```

### 过渡动画

```
淡入淡出: 300ms ease-in-out
滑动: 300ms cubic-bezier(0.4, 0, 0.2, 1)
缩放: 200ms ease-out
```

### 加载状态

```
加载中: 显示骨架屏或加载动画
加载成功: 内容淡入300ms
加载失败: 显示重试按钮
空状态: 显示插图 + 提示文字
```

## 💡 设计技巧

### 使用妙多AI的技巧

1. **提示词越详细越好**: 包括具体的尺寸、颜色、字号
2. **分步骤生成**: 先生成整体布局，再细化每个部分
3. **利用变体**: 为组件创建不同状态的变体
4. **使用样式系统**: 创建后统一应用，方便批量修改
5. **AI生成后优化**: AI生成是起点，需要人工精细调整

### 提高效率的方法

1. **复用组件**: 相似的组件（如卡片）在多个页面复用
2. **批量操作**: 使用妙多的批量编辑功能
3. **快捷键**: 熟练使用妙多的快捷键
4. **组件库**: 将常用组件保存到组件库
5. **团队协作**: 使用妙多的团队协作功能

## 📞 技术支持

### 常见问题

**Q: 设计稿尺寸为什么是375×812pt？**
A: 这是微信小程序的标准设计稿尺寸，对应750rpx宽度，便于开发适配。

**Q: 为什么要提供@2x和@3x切图？**
A: @2x适配iPhone SE等设备，@3x适配iPhone 16系列等高清屏设备。

**Q: 如何处理Dynamic Island？**
A: 使用`env(safe-area-inset-top)`自动适配，顶部预留59px安全区域。

**Q: TabBar页面需要底部安全区域吗？**
A: 不需要，TabBar会自动处理底部安全区域。

**Q: 妙多AI生成的设计如何优化？**
A: AI生成后需要检查：(1)尺寸是否精确 (2)间距是否符合8px网格 (3)颜色是否使用样式 (4)交互状态是否完整。
