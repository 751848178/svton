  alert events, dashboards, silences, notification channels, and notification
  deliveries through focused controllers/services; `MonitoringService` owns
  only alert evaluation orchestration.
- Organization map: focused monitoring controllers and `MonitoringService`
  currently import DTO classes through `./dto/monitoring.dto`; focused DTO
  files own domain-specific input classes, while `monitoring.dto.ts` remains
  the stable public barrel. `MonitoringAccessService` centralizes controller
  read/write access-policy wrapping. `MonitoringSchedulerService` owns
  scheduler orchestration; `MonitoringSchedulerConfigService` owns scheduler
  env/config interpretation; scheduler summary types and default summary value
  construction live in focused scheduler files. The resource metric dashboard
  and service SLO dashboard read-models now live behind focused query/builder
  services, and the dashboard controller calls those focused services plus the
  Service SLO template service directly for dashboard/template read routes.
  Notification-channel, notification-delivery, and silence HTTP routes now
  call focused channel/read/retry/silence services directly while preserving
  controller access filtering and write guards.
  Notification delivery payload/text/email formatting now lives behind focused
  builder/types/utils boundaries. Notification delivery dispatch/config/write
  execution now lives behind focused dispatch, adapter, writer, and config
  services, notification retry orchestration/audit now lives behind focused
  retry services, alert escalation scan/dedupe/audit now lives behind focused
  escalation services, alert silence CRUD/window/matching now lives behind
  focused silence services, alert event HTTP routes now call the focused event
  service directly, and alert event list/access/acknowledge/audit now lives
  behind focused event services. Alert rule CRUD/access and create-time
  target resolution now live behind focused rule services. Post-evaluation
  alert event persistence/rule status/audit/notification dispatch now lives
  behind a focused evaluation event service. Site certificate/TLS condition
  evaluation now lives behind focused Site certificate services, observed
  value plus generic target status evaluation now lives behind a focused status
  evaluation service, Service SLO breach/error-budget/exhaustion evaluation
  now lives behind focused SLO condition, signal, window, and metric evaluator
  services, Site/Deployment smoke-check failure evaluation now lives behind
  focused smoke-check evaluation services, log-count alert evaluation now lives
  behind a focused log-count evaluation service, generic Deployment/Backup
  status evaluation now lives behind focused status evaluation services, and
  cloud-provider sync plus resource metric threshold evaluation now live behind
  focused resource alert evaluation services.

  scheduler retry/escalation now calls focused services directly;
  category/metric alert evaluation dispatch, dashboard/template read routes,
  notification HTTP routes, and Service SLO rule template output now live
  behind focused services.
